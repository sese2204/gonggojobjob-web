# Feature Specification: 대외활동 검색 및 북마크 통합

**Feature Branch**: `002-activity-search`  
**Created**: 2026-04-10  
**Status**: Draft

### Session 2026-04-10

## Overview

취업 준비생이 채용공고뿐 아니라 대외활동(공모전, 봉사활동, 인턴십 등)도 검색하고 북마크할 수 있는 기능.
메인 페이지(`/`)의 검색 폼 상단에 "채용공고 / 대외활동" 탭이 있어 자연스럽게 `/activities`로 전환되며, 기존 채용공고 검색과 동일한 사용 패턴을 따른다.
북마크 시스템이 JOB/ACTIVITY 타입으로 확장되어 내 지원 현황 페이지에서 통합 관리된다.

---

## User Scenarios & Testing

### User Story 1 - 대외활동 검색 (Priority: P1)

사용자가 `/activities` 페이지에서 관심 태그와 키워드로 대외활동(공모전, 봉사활동 등)을 검색한다.
검색 결과에는 AI 매칭률과 코멘트가 포함되며, 기존 채용공고 검색과 동일한 UX를 제공한다.

**Why this priority**: 핵심 신규 기능. 대외활동 검색 자체가 가능해야 나머지 기능(북마크, 기록)이 의미를 가진다.

**Independent Test**: `/activities` 접속 -> 태그 선택 + 키워드 입력 -> 검색 -> 결과 목록 확인 (매칭률, 주최기관, 기간 등)

**Acceptance Scenarios**:

1. **Given** 사용자가 `/activities` 페이지에 접근하면, **When** 태그를 선택하고 키워드를 입력한 후 검색하면, **Then** 대외활동 목록이 매칭률순으로 표시되고, 각 항목에 제목/주최기관/카테고리/기간/설명/매칭률/AI코멘트가 포함된다.
2. **Given** 비로그인 사용자가 검색하면, **When** 하루 3회 제한에 도달하지 않았으면, **Then** 정상적으로 검색 결과가 표시된다.
3. **Given** 로그인 사용자가 하루 5회 검색을 모두 사용했으면, **When** 추가 검색을 시도하면, **Then** "오늘 검색 횟수를 모두 사용했어요" 안내가 표시된다.
4. **Given** 검색 결과에 "오늘 새로 올라온 활동" 수가 있으면, **Then** 별도로 강조 표시된다.

---

### User Story 2 - 메인에서 대외활동으로 자연스러운 전환 (Priority: P1)

메인 페이지(`/`)에서 대외활동 검색 페이지(`/activities`)로 쉽게 이동할 수 있는 진입점이 제공된다.
풋터에 대외활동 통계도 함께 표시된다.

**Why this priority**: 기존 사용자 동선에서 신규 기능을 자연스럽게 발견해야 실제 사용으로 이어진다.

**Independent Test**: 메인 페이지에서 대외활동 진입점 클릭 -> `/activities`로 라우팅 -> 활동 검색 가능 확인

**Acceptance Scenarios**:

1. **Given** 사용자가 메인 페이지 검색 폼에 있을 때, **When** "대외활동" 탭을 클릭하면, **Then** `/activities` 페이지로 이동한다.
2. **Given** 메인 페이지 풋터에서, **Then** 기존 채용공고 통계 옆에 대외활동 통계(총 활동 수, 오늘 새로 올라온 활동 수)가 표시된다.
3. **Given** 네비게이션 바에서, **Then** 대외활동 페이지로 이동할 수 있는 링크가 있다.

---

### User Story 3 - 대외활동 북마크 및 통합 관리 (Priority: P1)

사용자가 대외활동 검색 결과나 추천 활동에서 관심 활동을 "내 공고에 저장하기"로 북마크한다.
내 지원 현황 페이지(`/bookmarks`)에서 채용공고와 대외활동을 타입별로 필터링하여 관리한다.

**Why this priority**: 검색만으로는 가치가 제한적. 북마크/관리까지 되어야 실질적 가치를 제공한다.

**Independent Test**: 활동 검색 결과에서 저장 -> `/bookmarks?type=ACTIVITY`에서 확인 -> 타입 필터 전환

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 활동 검색 결과를 보고 있을 때, **When** "내 공고에 저장하기" 버튼을 클릭하면, **Then** 해당 활동이 ACTIVITY 타입으로 북마크된다.
2. **Given** 이미 저장한 활동을 다시 저장 시도하면, **Then** "이미 저장된 공고입니다" 메시지가 표시된다.
3. **Given** 내 지원 현황 페이지에서, **When** 상태 필터 위에 배치된 타입 필터(전체/채용공고/활동)를 선택하면, **Then** 해당 타입의 북마크만 표시되며, 상태 필터와 조합하여 사용할 수 있다.
4. **Given** 활동 타입 북마크 카드에는, **Then** 카테고리, 시작일, 종료일이 추가로 표시된다.
5. **Given** 비로그인 사용자가 활동 저장을 시도하면, **Then** 로그인 유도 안내가 표시된다.

---

### User Story 4 - 활동 검색 기록 및 추천 활동 관리 (Priority: P2)

로그인한 사용자가 과거 활동 검색 기록을 확인하고, AI 추천 활동 목록을 관리한다.
기존 채용공고의 "내 추천 공고" 패턴과 동일한 UX를 `/activities` 페이지에서 제공한다.

**Why this priority**: 핵심 검색+북마크 흐름이 먼저 완성되어야 하지만, 로그인 사용자의 재방문 가치를 높이는 중요 기능.

**Independent Test**: 활동 검색 수행 -> `/activities` 재방문 -> 추천 활동 목록 확인 -> 정렬/삭제/북마크

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 `/activities`에 접근하면, **When** 과거 검색 기록이 있으면, **Then** 추천 활동 목록이 매칭률순으로 기본 표시되고, "새로 검색하기" 버튼으로 검색 폼으로 전환할 수 있다.
2. **Given** 추천 활동 목록에서, **When** 삭제 버튼을 클릭하면, **Then** 해당 추천 활동이 목록에서 제거된다.
3. **Given** 추천 활동에서, **When** "내 공고에 저장하기"를 클릭하면, **Then** recommendedActivityId로 북마크가 생성된다.

---

### Edge Cases

- 활동의 종료일(endDate)이 이미 지난 경우? -> 검색 결과에 포함되되, 만료 표시를 함께 제공한다 (서버 응답 데이터 기반).
- 채용공고와 대외활동을 동시에 검색하고 싶은 경우? -> 각각 별도 페이지(`/`, `/activities`)에서 독립 검색. 통합 검색은 범위 밖.
- 기존 북마크(type 없음)와의 하위 호환? -> 기존 북마크는 type: "JOB"으로 취급. category/startDate/endDate는 null.
- 활동 검색 결과에서 외부 URL 클릭 시? -> 새 탭에서 원본 활동 페이지로 이동.
- 활동과 채용공고에서 검색 횟수가 공유되는가? -> 각각 독립적으로 하루 5회(로그인)/3회(비로그인) 제한.

## Requirements

### Functional Requirements

- **FR-001**: 시스템은 `/activities` 경로에서 대외활동 전용 검색 페이지를 제공해야 한다
- **FR-002**: 사용자는 태그와 키워드로 대외활동을 검색할 수 있어야 한다 (`POST /api/activities/search`)
- **FR-003**: 검색 결과 각 항목에는 id, title, organizer, category, startDate, endDate, description, url, match(매칭률), reason(AI 코멘트)가 포함되어야 한다
- **FR-004**: 검색 횟수는 비로그인 하루 3회, 로그인 하루 5회로 제한되어야 한다 (채용공고 검색과 독립)
- **FR-005**: 메인 페이지 검색 폼 상단에 "채용공고 / 대외활동" 탭 전환 UI가 있어야 하며, "대외활동" 탭 선택 시 `/activities`로 이동해야 한다
- **FR-006**: 네비게이션 바에 대외활동 링크가 추가되어야 한다
- **FR-007**: 로그인 사용자는 활동 검색 결과에서 activityListingId로, 추천 활동에서 recommendedActivityId로 북마크할 수 있어야 한다
- **FR-008**: 내 지원 현황 페이지에서 타입 필터(전체/JOB/ACTIVITY)가 상태 필터 위에 독립 배치되어야 하며, 두 필터를 조합하여 사용할 수 있어야 한다
- **FR-009**: ACTIVITY 타입 북마크 카드에는 카테고리, 시작일, 종료일이 표시되어야 한다
- **FR-010**: 로그인 사용자는 활동 검색 기록을 조회할 수 있어야 한다 (`GET /api/activity-search-history`)
- **FR-011**: 로그인 사용자는 추천 활동 목록을 매칭률순으로 조회하고 삭제할 수 있어야 한다
- **FR-012**: 풋터 통계에 대외활동 공고 수(activityTotalCount)와 오늘 새로 올라온 활동 수(activityNewTodayCount)가 표시되어야 한다

### Key Entities

- **Activity**: 대외활동 정보. id, title, organizer, category, startDate, endDate, description, url, match, reason
- **Bookmark (확장)**: 기존 북마크에 type("JOB"|"ACTIVITY"), activityListingId, category, startDate, endDate 필드 추가
- **ActivitySearchHistory**: 활동 검색 기록. 검색 조건(tags, query)과 추천 활동 목록 포함
- **RecommendedActivity**: AI 추천 활동. matchScore, reason 등 포함

## Success Criteria

### Measurable Outcomes

- **SC-001**: 사용자가 메인 페이지에서 2클릭 이내에 대외활동 검색 페이지에 도달할 수 있다
- **SC-002**: 대외활동 검색 결과가 채용공고 검색과 동일한 시간(체감 1초 이내) 내에 표시된다
- **SC-003**: 내 지원 현황에서 타입 필터 전환 시 0.5초 이내에 목록이 갱신된다
- **SC-004**: 기존 채용공고 북마크가 신규 타입 필드 추가 후에도 정상적으로 표시된다 (하위 호환 100%)
- **SC-005**: 모든 에러 상황(네트워크, 중복 저장, 검색 횟수 초과, 인증 필요)에 사용자 친화적 메시지가 표시된다

## Assumptions

- 대외활동 검색 태그는 프론트엔드에 `ACTIVITY_TAG_DATA`로 하드코딩하며, 채용공고 직무 태그와는 별도의 대외활동 맞춤 카테고리를 사용한다 (예: 공모전, 봉사활동, 서포터즈, 인턴십 등 활동 유형 중심).
- 활동 검색 횟수 제한은 채용공고 검색과 독립적으로 관리된다 (별도 localStorage 키 사용).
- 기존에 저장된 북마크는 모두 type: "JOB"으로 간주되며, 서버에서 마이그레이션을 처리한다.
- 활동 검색 결과의 정렬은 서버에서 매칭률순으로 제공되며, 클라이언트에서 추가 정렬은 불필요하다.
