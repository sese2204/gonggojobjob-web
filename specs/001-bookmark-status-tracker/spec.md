# Feature Specification: 공고 북마크 + 지원 상태 관리

**Feature Branch**: `001-bookmark-status-tracker`  
**Created**: 2026-04-08  
**Status**: Draft

## Overview

취업 준비생이 관심 공고를 저장(북마크)하고, 지원 상태를 추적할 수 있는 기능.
서비스 내 검색/추천 공고뿐 아니라 외부 공고(링크드인, 사람인 등)도 직접 입력하여 관리할 수 있다.
원본 공고가 만료/삭제되어도 저장된 정보는 유지된다.

---

## User Scenarios & Testing

### User Story 1 - 검색 결과에서 공고 저장 (Priority: P1)

사용자가 검색 결과 카드에서 "내 공고에 저장하기" 버튼을 눌러 관심 공고를 저장한다.
단방향 저장 방식으로, 카드에서는 저장만 가능하고 삭제는 "내 지원 현황" 페이지에서 수행한다.

**Why this priority**: 서비스의 핵심 가치. 검색 → 저장 흐름이 가장 빈번한 사용 패턴.

**Independent Test**: 로그인 후 검색 실행 → 결과 카드의 "내 공고에 저장하기" 버튼 클릭 → 성공 피드백 → 내 지원 현황에서 확인

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 검색 결과를 보고 있을 때, **When** 공고 카드의 "내 공고에 저장하기" 버튼을 클릭하면, **Then** 해당 공고가 저장되고 버튼 텍스트가 "저장 완료"로 변경된다 (2~3초 후 원복).
2. **Given** 이미 저장한 공고를 다시 저장 시도하면, **When** 서버가 409를 반환하면, **Then** "이미 저장된 공고입니다" 메시지가 표시된다.
3. **Given** 비로그인 사용자가 저장 버튼을 클릭하면, **Then** 로그인 유도 안내가 표시된다.

---

### User Story 2 - 북마크 목록 조회 및 상태 관리 (Priority: P1)

사용자가 "내 지원 현황" 페이지에서 저장한 공고 목록을 보고, 지원 상태를 업데이트한다.

**Why this priority**: 북마크의 핵심 활용 화면. 목록이 없으면 저장 기능의 의미가 없다.

**Independent Test**: `/bookmarks` 라우트로 직접 접근 → 북마크 목록 확인 → 상태 변경 → 메모 입력 → 삭제

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 내 지원 현황 페이지에 접근하면, **When** 페이지가 로드되면, **Then** 북마크한 공고 목록이 최신순으로 표시된다.
2. **Given** 북마크 목록에서 공고 카드를 볼 때, **When** 상태 드롭다운을 클릭하면, **Then** 5가지 상태(미지원/지원완료/면접 진행/불합격/합격) 중 선택하여 변경할 수 있다.
3. **Given** 공고 카드에서, **When** 메모 영역을 클릭하여 텍스트를 입력하고 포커스를 벗어나면, **Then** 메모가 자동 저장된다 (인라인 편집).
4. **Given** 북마크 목록에서, **When** 삭제 버튼을 클릭하면, **Then** 확인 후 해당 북마크가 삭제된다.
5. **Given** 상태 필터 탭을 선택하면, **When** 특정 상태를 클릭하면, **Then** 해당 상태의 공고만 필터링되어 표시된다.

---

### User Story 3 - 커스텀 공고 직접 추가 (Priority: P2)

사용자가 외부 채용 사이트에서 본 공고를 직접 입력하여 북마크 목록에 추가한다.

**Why this priority**: 서비스 외부 공고까지 통합 관리하는 차별화 기능이지만, 핵심 북마크 흐름이 먼저 완성되어야 한다.

**Independent Test**: 내 지원 현황 페이지에서 "공고 추가" 버튼 클릭 → 모달에서 정보 입력 → 저장 → 목록에 표시 확인

**Acceptance Scenarios**:

1. **Given** 내 지원 현황 페이지에서, **When** "공고 추가" 버튼을 클릭하면, **Then** 커스텀 공고 입력 모달이 열린다.
2. **Given** 모달에서 제목과 회사명을 입력하고, **When** 저장 버튼을 클릭하면, **Then** 커스텀 공고가 북마크 목록에 추가된다.
3. **Given** 모달에서 제목 또는 회사명이 비어있을 때, **When** 저장 버튼을 클릭하면, **Then** 필수 입력 안내 메시지가 표시된다.

---

### User Story 4 - 추천 공고에서 저장 (Priority: P2)

사용자가 AI 추천 공고 목록에서 관심 공고를 "내 공고에 저장하기" 버튼으로 저장한다.

**Why this priority**: 추천 공고도 주요 사용 흐름이나, 검색 결과와 동일한 패턴이므로 구현 우선순위는 낮다.

**Independent Test**: 내 추천 공고 목록에서 "내 공고에 저장하기" 버튼 클릭 → 내 지원 현황에서 확인

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 추천 공고 목록을 보고 있을 때, **When** 공고 카드의 "내 공고에 저장하기" 버튼을 클릭하면, **Then** 해당 공고가 `recommendedJobId`로 저장된다.
2. **Given** 이미 저장된 추천 공고를 다시 저장 시도하면, **Then** "이미 저장된 공고입니다" 메시지가 표시된다.

---

### Edge Cases

- 이미 저장한 공고를 다시 저장하면? → 409 응답을 처리하여 "이미 저장된 공고입니다" 안내
- 원본 공고가 삭제/만료되면? → 북마크 데이터는 스냅샷이므로 영향 없음
- 네트워크 오류 시? → 에러 토스트 표시, 버튼 상태 원복 (optimistic update rollback)
- 무한 스크롤 끝에 도달하면? → 더 이상 로드할 데이터 없음을 표시
- 메모가 500자를 초과하면? → 클라이언트에서 입력 제한, 서버 400 응답도 처리

## Requirements

### Functional Requirements

- **FR-001**: 시스템은 로그인한 사용자가 검색 결과 공고를 북마크할 수 있어야 한다 (`POST /api/bookmarks { jobListingId }`)
- **FR-002**: 시스템은 로그인한 사용자가 추천 공고를 북마크할 수 있어야 한다 (`POST /api/bookmarks { recommendedJobId }`)
- **FR-003**: 사용자는 외부 공고를 직접 입력하여 북마크에 추가할 수 있어야 한다 (`POST /api/bookmarks/custom`)
- **FR-004**: 사용자는 북마크 목록을 무한 스크롤로 조회할 수 있어야 한다 (`GET /api/bookmarks`, 내부적으로 페이지네이션)
- **FR-005**: 사용자는 지원 상태별로 북마크를 필터링할 수 있어야 한다 (`GET /api/bookmarks?status=`)
- **FR-006**: 사용자는 북마크의 지원 상태를 변경할 수 있어야 한다 (`PATCH /api/bookmarks/{id}`)
- **FR-007**: 사용자는 북마크에 메모를 추가/수정/삭제할 수 있어야 한다 (`PATCH /api/bookmarks/{id}`)
- **FR-008**: 사용자는 북마크를 삭제할 수 있어야 한다 (`DELETE /api/bookmarks/{id}`)
- **FR-009**: 검색 결과/추천 공고 카드에 "내 공고에 저장하기" 버튼이 있어야 하며, 중복 저장 시 안내 메시지를 표시해야 한다
- **FR-010**: 비로그인 사용자가 북마크를 시도하면 로그인을 유도해야 한다

### Key Entities

- **Bookmark**: 사용자가 저장한 공고 정보. id, jobListingId(nullable), title, company, url, description, status, memo, bookmarkedAt
- **Status**: 지원 상태 열거형. NOT_APPLIED | APPLIED | INTERVIEWING | REJECTED | OFFERED

---

## Success Criteria

### Measurable Outcomes

- **SC-001**: 사용자가 검색 결과에서 1클릭으로 공고를 북마크할 수 있다
- **SC-002**: 내 지원 현황 페이지에서 북마크 목록이 1초 이내에 로드된다
- **SC-003**: 상태 변경이 즉시 UI에 반영되고 서버와 동기화된다
- **SC-004**: 커스텀 공고 추가가 모달에서 3단계(입력 → 확인 → 저장) 이내로 완료된다
- **SC-005**: 모든 에러 상황(네트워크, 중복, 인증)에 사용자 친화적 메시지가 표시된다
